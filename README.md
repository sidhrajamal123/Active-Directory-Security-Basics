# Active Directory Security Lab

## Project Overview
This project demonstrates fundamental Active Directory (AD) administration and security concepts in a Windows domain environment. The focus was on understanding identity management, Group Policy Objects (GPO), access control, and security hardening techniques commonly used in enterprise environments.

---

## Objective
The primary objective of this lab was to gain practical exposure to Active Directory concepts and understand how centralized identity and access management improves organizational security.

Goals:
- Understand domain-based authentication
- Learn user and group management
- Explore Organizational Units (OUs)
- Review Group Policy configurations
- Apply basic security hardening concepts

---

## Lab Environment

### Operating System
- Windows Server
- Windows Client Machine

### Technologies Used
- Active Directory Domain Services (AD DS)
- Group Policy Management Console (GPMC)
- Local Security Policy
- Windows Administrative Tools

---

## Tasks Performed

### 1. Active Directory Domain Understanding
Reviewed how domain-based environments function and how centralized authentication works.

Activities:
- Explored domain architecture
- Understood domain controller responsibilities
- Reviewed authentication concepts
- Learned the importance of centralized management

Key Learning:
Active Directory provides centralized authentication and authorization for enterprise systems.

---

### 2. User and Group Management
Reviewed identity management concepts inside Active Directory.

Activities:
- Created and managed user accounts
- Reviewed password reset procedures
- Understood security groups
- Learned access control principles

Security Importance:
Proper user management reduces unauthorized access risks and improves accountability.

---

### 3. Organizational Units (OUs)
Explored how OUs help structure enterprise environments.

Activities:
- Reviewed logical grouping of users and systems
- Understood delegation concepts
- Examined policy inheritance

Key Learning:
OUs simplify administration and policy management.

---

### 4. Group Policy Management (GPO)
Reviewed security policies used to strengthen endpoint security.

Policies Studied:
#### Password Security Policy
- Password complexity requirements
- Minimum password length
- Password expiration settings

#### Account Lockout Policy
- Failed login attempt threshold
- Lockout duration settings

#### Endpoint Restrictions
- Security configuration review
- Administrative control concepts

Security Impact:
Group Policy enables centralized security management and standardization across systems.

---

### 5. Security Hardening Concepts
Reviewed techniques used to reduce attack surface.

Areas Covered:
- Least Privilege Principle
- Strong password enforcement
- Restricting unnecessary permissions
- Security baseline concepts

---

## Security Best Practices Learned
- Principle of Least Privilege (PoLP)
- Strong password enforcement
- Account monitoring
- Centralized identity management
- Security policy standardization
